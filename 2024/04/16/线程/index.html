多线程常用方法

public void run() 线程的任务方法
public void start() 启动线程
public String getName() 获得当前线程的名称
public void setName(String name) 为线程设置名称
public static Thread currentThread() 获取当前执行的线程对象
public static void sleep(long time) 让当前的线程休眠多少毫秒
public final void join() 让调用当前这个方法的线程先执行完

线程池

  1. 七大参数

    corePoolSize：指定线程池的核心线程的数量

    maximumPoolSize：指定线程池的最大线程数量

    keepAliveTime：指定线程池的存活时间

    unit：指定临时线程存活的时间单位

    workQueue：指定线程池的任务队列

    threatFactory：指定线程池的线程工厂

    handler：指定线程池 大人物拒绝策略

  2. 代码实现

  3. Runnable任务

    1. 方法

      execute（Runnable command）：执行Runnable任务

      Future submit(Callable task):执行Callable任务，返回未来任务对象，用于获取线程返回的结果

      void shutdown():等全部任务执行完毕后，在关闭线程池

      List shutdownNow():立刻关闭线程池，停止正在执行的任务，并返回队列未执行的任务

    2. 代码

  4. 线程池工具类

    1. 方法

      public static ExecutorService newFixedThreadPool(int nThreads):创建固定线程数量的线程池，如果某个线程因为执行异常而结束，那么线程池会补充一个新线程替代它

      public static ExecutorService newSingleThreadExecutor():创建只有一个线程的线程池对象，如果该线程出现异常而结束，那么线程池会补充一个新线程

      public static ExecutorService newCashedThreadPool()线程数量随着任务增加而增加，如果线程任务执行完毕而空闲了60s则会被回收掉

      public static ScheduledExecutorService new ScheduledThreadPool(int corePoolSize)创建一个线程池，可以实现在给定的延迟后执行任务或者定期执行任务

    2. 代码

    3. 补充

      Executors创建线程池虽然好用，但并不推荐使用，为了规避支援耗尽的风险，因此往往推荐使用ThreadPoolExecutor的方式
